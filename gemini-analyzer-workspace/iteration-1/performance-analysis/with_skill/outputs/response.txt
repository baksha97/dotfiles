### 1. Installation Speed & Parallelization (Critical)

**Bottleneck:** `setup-linux.sh` installs tools (lazygit, fzf, yq, just, uv, etc.) one by one. Each tool requires a GitHub API call to fetch the latest version, followed by a download and extraction.

**Optimization Strategy:**
*   **Parallel Downloads:** Use `xargs -P` or a background process pattern to fetch binaries in parallel.
*   **Batch API Queries:** Instead of calling the GitHub API 5+ times, use a single `gh api` or `curl` call with a GraphQL query to get all versions at once.

**Before (setup-linux.sh):**
```bash
# Sequential downloads
LG_VERSION=$(curl -s "..." | jq -r .tag_name)
curl -fsSLo /tmp/lazygit.tar.gz "..."
# ... repeat for fzf, yq, etc.
```

**After (Optimized Pattern):**
```bash
# Fetch versions in parallel using a single GraphQL query or background jobs
fetch_tool() {
  local name=$1 version_url=$2 download_url=$3
  if ! command -v "$name" &>/dev/null; then
    version=$(curl -s "$version_url" | jq -r .tag_name)
    curl -fsSLo "/tmp/$name" "${download_url//VERSION/$version}" & 
  fi
}
# Trigger parallel downloads
fetch_tool "lazygit" "api...lazygit" "download...VERSION..."
fetch_tool "fzf" "api...fzf" "download...VERSION..."
wait # Wait for all background downloads to finish
```

### 2. Network Efficiency: The Nerd Font Loop

**Bottleneck:** The loop in `setup-linux.sh` installs 8 fonts sequentially. Each `curl` call waits for the previous download and extraction to finish.

**Optimization:**
Use `xargs` with the `-P` flag to download and extract fonts in parallel.

```bash
# Optimized Nerd Font Install
printf "%s\n" "${nerd_fonts[@]}" | xargs -I {} -P 4 bash -c "
  font={}
  if [[ ! -f '$font_dir/.installed-\${font}-$nerd_fonts_version' ]]; then
    curl -fsSL '.../\${font}.tar.xz' | tar -xJ -C '$font_dir'
    touch '$font_dir/.installed-\${font}-$nerd_fonts_version'
  fi"
```

### 3. I/O & Redundancy: Stow and Find

**Bottleneck:** `setup-common.sh` calls `stow` multiple times and runs a global `find` command.
*   `find . -name .DS_Store -delete`: This traverses the entire repository on every setup.
*   Multiple `stow` calls: Each call re-scans directories.

**Optimization:**
*   **Combined Stow:** Invoke `stow` once for all packages: `stow -d stow -t "$HOME" zsh powerlevel10k tmux alacritty --adopt`.
*   **Targeted Cleanup:** Only delete `.DS_Store` files in the `stow/` directory, or better yet, add them to `.stow-local-ignore`.

### 4. Shell Startup Performance (Zsh)

**Bottleneck:** Sourcing every file in `.zshrc.d/` sequentially.
*   `nvm.zsh` and `sdkman-init.sh` are notorious for adding 200ms-500ms to startup.
*   `homebrew.zsh` likely calls `brew shellenv`, which is slow.

**Optimization Strategy:**
*   **Lazy Loading:** Do not initialize `nvm` or `sdkman` until the command (e.g., `node` or `java`) is actually used.
*   **Static Caching:** Cache the output of `brew shellenv` to a file and source that instead of executing the command every time.

**Before (.zshrc):**
```zsh
[[ -s "$HOME/.sdkman/bin/sdkman-init.sh" ]] && source "$HOME/.sdkman/bin/sdkman-init.sh"
```

**After (Lazy SDKMAN):**
```zsh
sdk() {
  unfunction sdk
  source "$HOME/.sdkman/bin/sdkman-init.sh"
  sdk "$@"
}
```

### 5. Idempotency & Resource Usage

**Bottleneck:** `setup-linux.sh` runs `apt-get update` every time, even if no packages need installing.
*   **Optimization:** Check if the package list has changed or if more than X hours have passed since the last update before running `apt-get update`.

**Bottleneck:** Redundant copies in `setup-common.sh` (`cp -R "$target" "$backup_dir/"`).
*   **Optimization:** Use `rsync -a` or only copy if the file is not already a symlink (avoiding backing up the same config repeatedly).
