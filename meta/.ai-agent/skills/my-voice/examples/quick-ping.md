## Quick Ping

Use this format for low-urgency messages: sharing a link, a brief FYI, a short follow-up, or a nudge. These are one-liners or close to it.

## Structure

```
[Optional: "(Not urgent)" or similar qualifier] - [link or brief message]
```

Or for follow-ups and discoveries:

```
:wave: Looks like [brief finding]. I've [action taken] here [link].

[Optional: 1 sentence of additional context or ask]
```

## Tone notes

- If it's not urgent, say so upfront — it sets expectations
- Link first, explain second (or not at all if the link is self-explanatory)
- These don't need `:wave:` unless they're going to someone outside the team

## Examples

```
(Not urgent) - https://github.com/org/repo/pull/4
```

```
:wave: looks like my discovery work for PROJ-71 helped identify specific implementation issues within the vendor SDK. I've opened up this issue on their public repository here with reproduction steps, code level details, a suggested solution, and requested additional requirements for automatic data reconciliation. IMO, wrt our code, we should just bump our vendor SDK version in the identity SDK and everything should just work once client teams bump up to that version.

(it's one of many issues we've opened for their SDK! :sweat_smile:... ref)

can we have this Github Issue attached to our official vendor support ticket @identity-eng? I'm happy to discuss further with anyone interested.
```
