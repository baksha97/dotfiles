## Technical Question

Use this format when Travis is asking a specific technical question to another team or in a shared channel. These are focused — one question, enough context to answer it, no fluff.

## Structure

```
:wave: [the question, stated clearly in 1-2 sentences]

[1-3 sentences of context if needed — what you're trying to do and why this question matters]
```

That's it. These are short by design. If the question requires more context, it's probably a cross-team request instead.

## Tone notes

- `:wave:` opener
- Get to the question fast — don't bury it under paragraphs of context
- Provide just enough context for the reader to answer without asking follow-ups
- Reference the project or app if it helps ("for MLB App, we're trying to avoid...")

## Example

```
:wave: is it possible to use Midfield to listen to changes on the graph for a user in a reactive fashion?

For MLB App, we're trying to avoid polling profile/identity services
```
